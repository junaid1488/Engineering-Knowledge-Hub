# CI/CD Interview Questions

## 1. What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Delivery/Deployment.

---

## 2. What is Continuous Integration (CI)?

Developers frequently merge code into a shared repository where automated builds and tests are executed.

---

## 3. What is Continuous Delivery (CD)?

Code is automatically prepared for release but requires manual approval before deployment.

---

## 4. What is Continuous Deployment?

Code is automatically deployed to production after passing all tests.

---

## 5. Difference Between Continuous Delivery and Continuous Deployment?

| Continuous Delivery | Continuous Deployment |
| ------------------- | --------------------- |
| Manual Approval     | Automatic Release     |
| Safer               | Faster                |

---

## 6. What are the Benefits of CI/CD?

* Faster Releases
* Automated Testing
* Reduced Errors
* Faster Feedback

---

## 7. What is a CI/CD Pipeline?

An automated workflow that builds, tests, and deploys applications.

---

## 8. What are the Stages of a CI/CD Pipeline?

1. Source
2. Build
3. Test
4. Deploy
5. Monitor

---

## 9. What is Pipeline as Code?

Defining CI/CD pipelines using code files.

Examples:

* Jenkinsfile
* GitHub Actions YAML

---

## 10. What is a Build?

The process of compiling and packaging source code.

---

## 11. What is Automated Testing?

Running tests automatically during the pipeline.

---

## 12. What is Artifact?

A build output generated by the pipeline.

Examples:

* JAR
* WAR
* Docker Image

---

## 13. What is Rollback?

Reverting an application to a previous stable version.

---

## 14. What is Blue-Green Deployment?

Two environments:

* Blue (Current)
* Green (New)

Traffic switches after validation.

---

## 15. What is Canary Deployment?

Releases a new version to a small group of users before full rollout.

---

## 16. What is GitHub Actions?

GitHub's built-in CI/CD platform.

---

## 17. What is Jenkins?

An open-source automation server used for CI/CD.

---

## 18. What is GitLab CI/CD?

Built-in CI/CD solution provided by GitLab.

---

## 19. What is a Webhook?

An event-based trigger used to start pipelines automatically.

---

### CI/CD Workflow

```text
Developer
    ↓
Git Push
    ↓
CI Pipeline
(Build + Test)
    ↓
Artifact

CD Pipeline
```
